---
title: Song Requests
description: Song request queue system for Twitch chat.
---

The song request system allows viewers to request songs via chat commands. Moderators can manage the queue, skip songs, and configure settings from the web dashboard.

## Setup

1. Navigate to **Dashboard > Song Requests** in the web dashboard
2. Enable song requests in the **Settings** card
3. Configure queue limits and access levels as needed

## Settings

| Setting | Default | Description |
|---------|---------|-------------|
| Enabled | Off | Whether song requests are active |
| Max Queue Size | 50 | Maximum number of songs in the queue |
| Max Per User | 5 | Maximum songs a single user can have queued |
| Min Access Level | Everyone | Minimum access level required to request songs |

## Chat Commands

### Viewer Commands

- `!sr <song title>` — Request a song (aliases: `!songrequest`, `!song`)
- `!sr list` / `!sr queue` — Show the next 5 songs in the queue
- `!sr current` — Show the currently playing song
- `!sr remove` — Remove all of your own song requests

### Moderator Commands

- `!sr skip` — Skip the current song
- `!sr remove <position>` — Remove a song at a specific position
- `!sr clear` — Clear the entire song request queue

## Dashboard Management

The **Song Requests** page in the web dashboard provides:

- **Settings card** — Toggle song requests on/off, configure queue size limits, per-user limits, and minimum access level
- **Queue table** — View all queued songs with position, title, requester, and time. Moderators can skip, remove individual songs, or clear the entire queue

## Access Level

The `!sr` command access level can be customized per-channel from the Commands page. The minimum access level for requesting songs is separately configured in the song request settings.
