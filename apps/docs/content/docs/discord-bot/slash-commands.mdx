## `/help [topic]`

Displays contextual help for any command group. If no topic is specified, shows an overview of all available commands.

**Parameters:**
- `topic` (optional) ‚Äî Choose from: twitch, quote, config, roles, template, mod, case, cc, report, data, schedule

## `/mod` ‚Äî Moderation

All `/mod` commands require the configured **Mod Role** or the **Manage Messages** Discord permission. Every action creates a numbered moderation case and logs to the configured moderation log channel.

### `/mod ban <user> [reason] [delete-days]`

Permanently ban a user from the server.

- `user` (required) ‚Äî The user to ban
- `reason` (optional) ‚Äî Reason for the ban
- `delete-days` (optional) ‚Äî Days of message history to delete (0‚Äì7)

### `/mod tempban <user> <duration> [reason]`

Temporarily ban a user. The bot does not auto-unban ‚Äî use `/mod unban` to lift.

- `user` (required) ‚Äî The user to ban
- `duration` (required) ‚Äî Duration in minutes
- `reason` (optional) ‚Äî Reason for the ban

### `/mod kick <user> [reason]`

Kick a user from the server.

- `user` (required) ‚Äî The user to kick
- `reason` (optional) ‚Äî Reason for the kick

### `/mod warn <user> <reason>`

Warn a user. If the user's active warning count hits a configured threshold (see `/config thresholds`), the escalation action (ban, kick, or mute) is automatically applied.

- `user` (required) ‚Äî The user to warn
- `reason` (required) ‚Äî Reason for the warning

### `/mod mute <user> <duration> [reason]`

Timeout a user using Discord's built-in timeout feature. Maximum duration: 28 days (40,320 minutes).

- `user` (required) ‚Äî The user to mute
- `duration` (required) ‚Äî Duration in minutes (1‚Äì40,320)
- `reason` (optional) ‚Äî Reason for the mute

### `/mod unban <user-id> [reason]`

Unban a previously banned user. Requires the user ID (not a mention, since banned users can't be mentioned).

- `user-id` (required) ‚Äî The user ID to unban
- `reason` (optional) ‚Äî Reason for the unban

### `/mod unmute <user> [reason]`

Remove a timeout from a user.

- `user` (required) ‚Äî The user to unmute
- `reason` (optional) ‚Äî Reason for removing the mute

### `/mod unwarn <user> [reason]`

Remove the most recent active warning from a user.

- `user` (required) ‚Äî The user to unwarn
- `reason` (optional) ‚Äî Reason for removing the warning

## `/case` ‚Äî Case Management

All `/case` commands require the configured **Mod Role** or the **Manage Messages** Discord permission. Results are paginated with navigation buttons.

### `/case lookup <number>`

Look up a specific moderation case by its number. Shows full details including type, target, moderator, reason, duration, resolution status, and notes.

- `number` (required) ‚Äî The case number to look up

### `/case list [user] [type]`

List recent moderation cases with optional filters. Shows up to 100 cases, 10 per page. Resolved cases are shown with strikethrough.

- `user` (optional) ‚Äî Filter cases for a specific user
- `type` (optional) ‚Äî Filter by case type (BAN, TEMPBAN, KICK, WARN, MUTE, UNBAN, UNWARN, UNMUTE, NOTE)

### `/case note <number> <content>`

Add a moderator note to a case.

- `number` (required) ‚Äî The case number
- `content` (required) ‚Äî The note content

### `/case search <query>`

Search cases by reason text (case-insensitive). Returns up to 50 results.

- `query` (required) ‚Äî Text to search for in case reasons

## `/config` ‚Äî Server Configuration

All `/config` commands require the configured **Admin Role** or the **Manage Server** Discord permission.

### Log Channels

#### `/config log set-moderation <channel>`

Set the channel where moderation actions (bans, kicks, warns, mutes, reports) are logged.

#### `/config log set-server <channel>`

Set the channel where server events (channel/role create/delete/update) are logged.

#### `/config log set-voice <channel>`

Set the channel where voice activity (joins, leaves, moves) is logged.

#### `/config log view`

View the current log channel configuration for all three categories.

### Warning Thresholds

Configure automatic escalation when a user reaches a certain number of active warnings.

#### `/config thresholds set <count> <action> [duration]`

Set an escalation action at a specific warning count. For example, "at 3 warnings, auto-ban."

- `count` (required) ‚Äî Warning count to trigger at (1‚Äì50)
- `action` (required) ‚Äî Action to take: BAN, KICK, or MUTE
- `duration` (optional) ‚Äî Duration in minutes (required for MUTE)

#### `/config thresholds clear <count>`

Remove a warning threshold.

- `count` (required) ‚Äî The warning count threshold to remove

#### `/config thresholds list`

List all configured warning thresholds.

## `/roles` ‚Äî Self-Assignable Roles

All `/roles` commands require the configured **Admin Role** or the **Manage Server** Discord permission. Panels support up to 25 roles each.

### Panel Management

#### `/roles panel create <name> [title] [description] [use-menu]`

Create a new role panel. By default, panels use buttons. Set `use-menu` to `true` for a dropdown select menu instead.

- `name` (required) ‚Äî Panel name (unique per guild)
- `title` (optional) ‚Äî Panel embed title (default: "Role Selection")
- `description` (optional) ‚Äî Panel embed description
- `use-menu` (optional) ‚Äî Use a select menu instead of buttons

#### `/roles panel delete <name>`

Delete a role panel and all its buttons. Autocomplete available.

#### `/roles panel list`

List all role panels with their mode (buttons/menu), post status, and assigned roles. Paginated, 5 panels per page.

### Button Management

#### `/roles button add <panel> <role> <label> [emoji] [style]`

Add a role button to a panel.

- `panel` (required) ‚Äî Panel name (autocomplete)
- `role` (required) ‚Äî The role to assign/remove on click
- `label` (required) ‚Äî Button label text
- `emoji` (optional) ‚Äî Button emoji
- `style` (optional) ‚Äî Button style: primary (blue), secondary (grey), success (green), danger (red)

#### `/roles button remove <panel> <role>`

Remove a role button from a panel.

### Posting

#### `/roles post <panel> [channel]`

Post a role panel to a channel. Users can click buttons or use the select menu to toggle roles.

- `panel` (required) ‚Äî Panel name (autocomplete)
- `channel` (optional) ‚Äî Target channel (defaults to current)

#### `/roles refresh <panel>`

Update an already-posted panel message with the current button/menu configuration.

## `/template` ‚Äî Message Templates

All `/template` commands require the configured **Admin Role** or the **Manage Server** Discord permission. Templates support text content, embed JSON, or both.

**Supported variables:** `{server}` (server name), `{user}` (command user), `{memberCount}` (server member count)

### `/template create <name> [content] [embed-json]`

Create a new message template. At least one of `content` or `embed-json` is required.

### `/template edit <name> [content] [embed-json]`

Edit an existing template. Autocomplete available for name.

### `/template delete <name>`

Delete a template. Autocomplete available.

### `/template list`

List all templates with pagination (10 per page). Shows whether each template has text, embed, or both.

### `/template preview <name>`

Preview a template with variables replaced in the current context.

### `/template send <name> [channel]`

Send a template to a channel with variable substitution.

## `/schedule` ‚Äî Scheduled Messages

All `/schedule` commands require the configured **Admin Role** or the **Manage Server** Discord permission.

### `/schedule create <name> <channel> <type> [content] [cron] [template]`

Create a scheduled message. Must provide either `content` or `template`.

- `name` (required) ‚Äî Schedule name
- `channel` (required) ‚Äî Target text channel
- `type` (required) ‚Äî `ONCE` (manual trigger) or `RECURRING` (cron-based)
- `content` (optional) ‚Äî Message content
- `cron` (optional) ‚Äî Cron expression (required for RECURRING, e.g., `0 9 * * *` for daily at 9 AM)
- `template` (optional) ‚Äî Template name to use (autocomplete)

### `/schedule edit <name> [content] [cron] [channel]`

Edit a scheduled message. Autocomplete available for name.

### `/schedule delete <name>`

Delete a scheduled message.

### `/schedule list`

List all scheduled messages (8 per page) with status, type, schedule, and target channel.

### `/schedule enable <name>` / `/schedule disable <name>`

Toggle a scheduled message on or off without deleting it.

## `/cc` ‚Äî Custom Commands

Custom commands are guild-scoped slash commands with dynamic responses.

- **Creating/editing/toggling** requires the **Mod Role** or **Manage Messages** permission
- **Deleting** requires the **Admin Role** or **Manage Server** permission
- **Running** is public (unless the command has role restrictions)

**Supported variables:** `{user}`, `{usermention}`, `{userid}`, `{server}`, `{channel}`, `{membercount}`

### `/cc create <name> <response> [description] [ephemeral]`

Create a custom command. Names are normalized to lowercase alphanumeric with hyphens (1‚Äì32 characters).

- `ephemeral` (optional) ‚Äî If true, responses are only visible to the user who ran the command

### `/cc edit <name> [response] [description] [ephemeral]`

Edit a custom command. Autocomplete available.

### `/cc delete <name>`

Delete a custom command (Admin only). Autocomplete available.

### `/cc list`

List all custom commands with status and use count.

### `/cc toggle <name>`

Enable or disable a custom command.

### `/cc run <name>`

Run a custom command. Checks role restrictions and enabled status.

## `/report` ‚Äî User Reports

Anyone can report a user. Managing reports requires the **Mod Role** or **Manage Messages** permission.

### `/report user <user> <reason>`

Report a user to the moderators. Cannot report yourself or bots. Creates a report with OPEN status and logs to the moderation log channel.

### `/report status <id> <status> [resolution]`

Update a report's status. Mod only.

- `id` (required) ‚Äî Report number
- `status` (required) ‚Äî INVESTIGATING, RESOLVED, or DISMISSED
- `resolution` (optional) ‚Äî Resolution notes

### `/report list [status] [target]`

List reports with optional filters. Mod only. Paginated, 8 per page.

- `status` (optional) ‚Äî Filter by OPEN, INVESTIGATING, RESOLVED, or DISMISSED
- `target` (optional) ‚Äî Filter by reported user

## `/data` ‚Äî Data Privacy

Available to all users. Allows exporting or deleting your personal data.

### `/data export`

Exports all data the bot has about you in this server as a JSON file. Includes moderation cases, reports (as reporter or target), and active warning count.

### `/data delete`

Deletes your data from the server. Reports where you are the reporter are deleted. Moderation cases (bans, warns, etc.) are **not** deleted as they are server moderation records.

## `/twitch` ‚Äî Twitch Notifications

All `/twitch` commands require the configured **Admin Role** or the **Manage Server** Discord permission.

**Prerequisites:** A notification channel must be set with `/twitch notifications set-channel` before live notifications will work.

### `/twitch add <username>`

Add a Twitch channel to monitor for live stream notifications. Validates the username against the Twitch API.

### `/twitch remove <username>`

Remove a Twitch channel from the notification list.

### `/twitch list`

List all monitored Twitch channels with live/offline status indicators.

### `/twitch test <username>`

Send a test notification to verify setup. **Bot owner only** (`OWNER_ID` env var). After 10 seconds, the embed updates to show offline state.

### `/twitch notifications set-channel <channel>`

Set the Discord channel for Twitch live notifications.

### `/twitch notifications set-role <role>`

Set a role to ping when a stream goes live.

## `/quote` ‚Äî Cross-Platform Quotes

Shared quote database with the Twitch bot. Adding and removing quotes requires the **Mod Role** or **Manage Messages** permission.

### `/quote show [number]`

Show a random quote, or a specific quote by number.

### `/quote add <text>`

Add a new quote.

### `/quote remove <number>`

Remove a quote by number.

### `/quote search <text>`

Search quotes by keyword.

## Configurable Role Mapping

By default, commands check standard Discord permissions:
- **Admin commands** ‚Äî Manage Server permission
- **Mod commands** ‚Äî Manage Messages permission

You can override this by configuring role mappings in the web dashboard under **Discord Settings > Role Mapping**:

- **Admin Role** ‚Äî Members with this role can use admin-level commands (`/twitch`, `/config`, `/roles`, `/template`, `/schedule`, `/cc delete`)
- **Mod Role** ‚Äî Members with this role can use mod-level commands (`/mod`, `/case`, `/report status/list`, `/cc create/edit/toggle`, `/quote add/remove`). Admin role holders also have mod permissions.
